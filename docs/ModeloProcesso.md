# Modelo de Processo e Plano de Trabalho

## 1. Introdução
Este documento descreve o modelo de processo e plano de trabalho para o desenvolvimento do Sistema de Agendamento de Compromissos, desenvolvido em três sprints conforme especificado.

## 2. Modelo de Processo
O projeto será desenvolvido utilizando uma abordagem ágil adaptada, com três sprints principais:

### Sprint 1: Planejamento e Especificação
**Duração**: 1 semana
**Atividades**:
1. Definição do modelo de processo
2. Criação do documento de especificação de requisitos
3. Definição dos requisitos funcionais e não-funcionais

**Responsáveis**:
- Scrum Master: Coordenação do sprint
- Product Owner: Definição dos requisitos
- Desenvolvedores: Análise técnica

### Sprint 2: Projeto Arquitetural
**Duração**: 1 semana
**Atividades**:
1. Detalhamento dos requisitos com histórias de usuário
2. Criação dos cenários de teste
3. Desenvolvimento dos diagramas UML:
   - Diagrama de Classes
   - Diagrama de Componentes
   - Diagrama de Sequência

**Responsáveis**:
- Arquiteto de Software: Criação dos diagramas
- Desenvolvedores: Revisão técnica
- QA: Definição dos cenários de teste

### Sprint 3: Implementação e Testes
**Duração**: 1 semana
**Atividades**:
1. Implementação parcial do sistema
2. Desenvolvimento dos testes automatizados
3. Documentação técnica
4. Preparação da apresentação

**Responsáveis**:
- Desenvolvedores: Implementação
- QA: Testes e validação
- Scrum Master: Coordenação da apresentação

## 3. Cronograma Detalhado

### Sprint 1 (Semana 1)
- **Dia 1**: Definição do modelo de processo
- **Dia 2-3**: Criação da especificação de requisitos
- **Dia 4-5**: Revisão e refinamento dos requisitos

### Sprint 2 (Semana 2)
- **Dia 1-2**: Criação das histórias de usuário
- **Dia 3-4**: Desenvolvimento dos diagramas UML
- **Dia 5**: Revisão e refinamento da documentação

### Sprint 3 (Semana 3)
- **Dia 1-3**: Implementação do sistema
- **Dia 4**: Desenvolvimento dos testes
- **Dia 5**: Preparação da apresentação

## 4. Ferramentas Utilizadas
- **Desenvolvimento**: Java, SQLite
- **Testes**: JUnit
- **Modelagem**: Draw.io, PlantUML
- **Controle de Versão**: Git
- **Documentação**: Markdown

## 5. Entregáveis
1. Documentação completa do projeto
2. Código fonte do sistema
3. Testes automatizados
4. Apresentação do projeto

## 6. Critérios de Aceitação
- Documentação completa e bem estruturada
- Código fonte funcional e bem documentado
- Testes automatizados com cobertura adequada
- Apresentação clara e objetiva do projeto
